# 如何提交專案

## 格式模板

```
#### 開發者名字(城市名) - [GitHub](), [部落格]()
* :white_check_mark: [產品名稱](網址):簡短的介紹,盡量把事情說清楚 - [更多介紹]()
```

城市名、GitHub、部落格、更多介紹均為選填。

## 格式模板 - 最精簡的版本(只保留必填項)

```
#### 開發者名字
* :white_check_mark: [產品名稱](網址):簡短的介紹,盡量把事情說清楚
```

## 如何寫好一句話介紹

**先定性,再加分**:開頭先用最短的詞說清楚產品是什麼,然後說讓你與眾不同的地方。

```
[產品類型],[核心價值或獨特之處]
```

參考範例:

- `AI 語音克隆工具,3 秒克隆任何聲音,支援中英日韓等 10 種語言,提供 50+ 預設聲音庫`
- `圖片壓縮工具,70%+ 壓縮率同時保持畫質,支援 PNG/JPEG/WebP 批次處理,純本地執行不上傳`
- `Chrome 書籤管理外掛,取代預設新分頁,支援資料夾卡片檢視、拖曳排序和批次操作`
- `Markdown 轉 Word/PDF 工具,支援 LaTeX 公式和 Mermaid 圖表,瀏覽器本地處理無需註冊`
- `Mac 選單列速記工具,全域懸浮不切換畫面,支援圖文混排和一鍵叫出剪貼簿歷史`
- `多人聚會地點推薦工具,輸入多人地址自動計算公平中點,免費無需註冊`
- `多語言聽力練習 App,支援匯入 Podcast 和 YouTube 連結,提供雙語字幕和跟讀練習`

沒有獨特之處時,加上具體數字或使用限制(免費、無需註冊、本地處理)也能讓介紹語更有價值。

## 收錄標準

1. **開發者必須是台灣人或台灣團隊**(個人或小型團隊的獨立開發,而非公司主力產品)
2. 主版面收錄「打開即用」的網站或 App;開發者工具請提交到[工程師版面](./pages/README-Programmer-Edition.md),遊戲請提交到[遊戲版面](./pages/README-Game.md)
3. 歡迎自薦或推薦他人作品,發 Pull Request 或 Issue 即可
